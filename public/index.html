SaaShup helps teams focus on product and revenue while the platform handles the SaaS engine around the application. This tool focuses on one operational part of that engine: controlled Docker image upgrades and instance lifecycle actions.

What it does

Simple operations for Docker-based SaaS instances

＋

Create

Create a instance, attach storage, configure network settings, publish DNS and prepare labels.

↻

Upgrade

Move matching containers from an old image version to a new one and wait for each recreate to complete.

⟳

Manage

Manage instances, images and version.

⌫

Delete

Remove an instance when it is no longer needed.

Workflow

From request to running instance

The admin page sends lifecycle requests, applies instance changes and exposes live activity logs for operators.

  1. SelectChoose destination, image, version and target instance.
  2. ExecuteTrigger create, upgrade, restart or delete.
  3. ObserveFollow the latest activity logs from the admin page.
